Version 0.2.1

September 3rd 2020

  • Added a disclaimer to the bottom of the contact form to let people know it won't actually do anything. Much like the rest of the site, it's just for show.
  • Added a top bar navigation that appears at non-mobile viewport widths.
  • This changelog was born! Took a little work to recall all of the changes made to the site before this existed.
  • Created a 404 page! Just in case something messes up for users.
  • Because multiple pages now exist, the internal stylesheet's been made external!
    • To make the site heavily responsive, a single-column layout was implemented via CSS grid. Really excited about this one tbh. The grid applies to the entire Main section of the site and utilizes named grid-lines, fractional margins, and a set of three center columns to place the content in.
    • All emphermal unit values for font sizes on the site have been replaced with calc functions that take emphermal units and viewport widths into account. This enables all text on the site to very responsively scale in size with relation to the size of the device and the user's personal taste in magnifcation and font size.
    • Implemented seperate 650px and 1024px breakpoints for tablet and desktop users, respectively.
      • At 650px, the top navigation bar appears and the margins of the CSS grid become a min-max value based on the Ch unit, which is responsive for users with varying viewport magnifications.
      • When below 650px, site navigation can be accessed from an on-page link-list near the footer of the page. Maybe if we get more pages and I become a more experienced developer I'll develop a hamburger menu for mobile users.
      • At 1024px, our fun social media links make their return!
    • Pageload animation! Most of the site's content enters with simple half-second fade-in and slide when the page is loaded.
    • The social media icons have their own seperate animations. A fun wipe and fade of the text-color and button background now occurs on and off of cursor hovers.
    • The fixed top navigation that occurs at non-mobile viewport-widths has a gentle scale-up animation on and off of mouse hover. It's very playful.
    • Wrote some specific CSS rules to help format this changelog page.

Known Issues

  • The animation on the social media link buttons that's supposed to occur when the mouse hovers off of the button happens to occur on pageload. There doesn't appear to be a simple solution to this that doesn't use Javascript, and hiding it under a loading animation would just be silly.
  • Gibson and Questa do not appear to load in on mobile devices. The site does not look good on Apple's default San Francisco family :/
  • The email contact form still doesn't do anything, but users should at least know that now thanks to the disclaimer implemented in this version.

Version 0.2

August 10th 2020

  • A complete rewrite of the previous version. All copy and code replaced with brand new stuff.
  • Rewrote metadata in the header to be more focused on myself.
  • Implemented an internal stylesheet that gives a completely new look to the website.
    • Created a large header at the top of the page that utilized a CSS grid to lay introductory content out into a 2x2 grid with internal box lines between them. Because of CSS grid, it responsively scaled with browser viewports.
    • Instead of the muted background color seen in 0.1, we've switched to a slightly Sprite-inspired color gradient that scales to the height and width of the viewport. With a fixed background attatchment, the body of the site appears to scroll over-top of this fixed background.
    • Switched to Gibson and Questa as the page's fonts. The Gibson family is used for 99% of content, with Questa used as a subtitle at the top of the page.
    • Switched to using emphermal units(em) as unit of measurement to determine the font sizes used on the site. In the previous version pixels(px) were used. Emphermal units are more appropriate as they provide better support for magnifcaiton of the viewport by users.
    • Implemented an SVG 'read more' arrow after the introductory section of the page.
    • Implemented a 650px breakpoint to make the site a bit more responsive in layout.
    • While about the 650px breakpoint, fun little social media links appear after the introduction CSS grid. The slightly experimental background-clip decleration is used to make the text of the links a fun little gradient when hovered over.
    • A stylized contact form was added near the bottom of the page! Note: Does not actually do anything on the backend.
    • Wrote a footer that begs people not to steal my identity or IP in any way. Be nice please.

Known Issues

  • The wrapper behind the media links has a larger hover state than the link itself. This can cause users to see the white background but not the gradient text. It looks bad when this happens.
  • Gibson and Questa do not appear to load in on mobile devices. The site does not look good on Apple's default San Francisco family :/
  • The email contact form does not actually do anything. Users have no way of knowing this.
